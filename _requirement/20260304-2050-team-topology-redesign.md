# 需求清单：Team Topology UI 完全重设计
> 创建时间：2026-03-04 20:50
> 状态：已确认

## 功能需求

1. **Leader 机制重设计** — 当前"根节点 = leader、leader 不可删除"的设计非常不灵活。用户添加了节点后想删掉重新排布，但 leader 删不掉，导致整个 team 废掉。需要重新思考 leader 机制，让拓扑图操作更灵活。
   > "这个team leader的这个设计就是很他妈的蠢...我又想把这个节点删除重新排布,但是我删除了sub agent,对于这个team leader agent我就没法删除...它一点都不灵活"

2. **修复节点重叠** — 点击 "+" 添加 agent 后，新节点会和已有节点重叠，交互不好。需要调研最佳实践来解决自动布局问题。
   > "我点击加号之后添加的agent它会覆盖到下一个agent,比如说第二个sub agent它会和第一个sub agent重叠,这种交互机制都不是最佳的实践"

3. **Team Instruction 交互重设计** — 当前点击展开/折叠的交互感觉像临时弹出，作为配置项很别扭，设计层面很差。需要重新设计为更自然的配置交互。
   > "这个team instruction的显示也非常的奇怪,它点击了之后然后可以展示,展示了之后再点击又折叠...这种配置都让人感觉的非常的别扭,这在设计层面上是很垃圾的"

4. **整体 UI 完全重新设计** — 当前实现又丑又难看又不好用，UI 布局奇怪。完全不参考当前实现，由 UI/UX 设计师从专业角度全面重新设计。
   > "可以完全不参考我们当前的实现,然后进行重新设计,当前的实现我觉得又丑又难看又不好用,然后UI布局也很奇怪,重新设计,完全重新设计"

## 流程要求

1. **联网调研最佳实践** — 在设计之前，先联网搜索 node-based topology/graph UI 的最佳实践，以调研结果为基础进行设计。
   > "我们联网搜索进行一定的调研,然后找出最佳实践是怎样的,然后再说这个设计"

## 风格要求

1. 遵循项目现有的 Minecraft / 像素艺术风格设计语言（Tailwind CSS v4, pixel fonts, no border-radius, shadow-pixel 等）
2. 暗色主题

## 注意事项

1. 可以完全不参考当前实现，从零重新设计
2. UI/UX 设计师是最核心的角色，由其主导视觉和交互设计
3. 必须先调研最佳实践再做设计
