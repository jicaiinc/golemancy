# 需求清单：Speech-to-Text（语音转文字）

> 创建时间：2026-02-24 16:00
> 状态：已确认

## 功能需求

1. **Chat 输入框增加 STT 能力**——麦克风按钮，录音后转写为文字回填到输入框
2. **只支持 OpenAI 兼容 API**——预置 OpenAI + Custom（OpenAI-compatible endpoint），用户填 baseURL + apiKey + 模型即可
3. **预置 Provider**：
   - OpenAI（baseURL 留空=默认），默认模型：`gpt-4o-mini-transcribe`、`gpt-4o-transcribe`、`whisper-1`
   - Custom OpenAI-Compatible（用户自填 baseURL + 模型）
4. **录音时实时波形动画**——像素方块柱状图风格，声音大波形高，声音小波形低
5. **音频持久化到服务端**——录音完成后上传保存，转写失败也不丢失音频
6. **手动重试**——转写失败时，用户在 UI 上手动点击重试（无自动重试、无 fallback provider）
7. **转写历史页面**——Global level 独立页面，查看所有转写记录：
   - 播放原始录音
   - 复制转写文本
   - 手动重试失败的记录
   - 删除单条记录
   - 清理历史 + 显示存储用量

## 配置设计

1. **Global Settings 新增 "Speech" Tab**——与 General、Providers Tab 平级
2. **STT Provider 独立配置**——不复用聊天模型 `providers` Record，单独管理
3. **配置项**：
   - Enable/Disable 开关
   - Provider 类型选择（OpenAI / Custom OpenAI-Compatible）
   - API Key
   - Base URL（Custom 时必填，OpenAI 时可选用于 proxy）
   - Model（下拉选择 + 可自定义输入）
   - Language（可选，默认自动检测）
   - Test Connection 按钮
4. **配置范围**：全局（App-level），初版不做 project-level override

## 技术约束

1. 服务端使用 Vercel AI SDK `experimental_transcribe` + `createOpenAI({ baseURL })` 实现转写
2. 前端录音使用原生 `MediaRecorder` API（Electron Chromium 完全支持）
3. 波形可视化使用 Web Audio API `AnalyserNode` + Canvas 2D 自绘（像素方块风格，不用第三方库）
4. macOS Electron 需通过主进程 `systemPreferences.askForMediaAccess('microphone')` 请求麦克风权限
5. 遵循现有项目技术栈约束（见 CLAUDE.md）：motion/react、react-router v7、Tailwind CSS v4、Hono、drizzle-orm 等

## 数据模型

1. **`SpeechToTextSettings`** — 新增到 `GlobalSettings`
   ```
   speechToText?: {
     enabled: boolean
     providerType: 'openai' | 'openai-compatible'
     apiKey?: string
     baseUrl?: string
     model: string
     language?: string
     testStatus?: 'untested' | 'ok' | 'error'
   }
   ```

2. **`TranscriptionRecord`** — 转写历史记录
   ```
   id: TranscriptionId (branded)
   createdAt: string (ISO)
   status: 'pending' | 'success' | 'failed'
   audioFileId: string
   audioDurationMs: number
   audioSizeBytes: number
   text?: string (转写文本)
   error?: string (错误信息)
   provider: string
   model: string
   projectId?: ProjectId
   conversationId?: ConversationId
   usedInMessage: boolean
   ```

3. **`ISpeechService`** — 独立服务接口
   ```
   transcribe(audio, config): Promise<TranscriptionRecord>
   listHistory(params?): Promise<TranscriptionRecord[]>
   getAudioUrl(audioFileId): string
   deleteRecord(id): Promise<void>
   clearHistory(): Promise<{ deletedCount: number; freedBytes: number }>
   retry(id): Promise<TranscriptionRecord>
   testProvider(config): Promise<{ ok: boolean; error?: string; latencyMs?: number }>
   getStorageUsage(): Promise<{ totalBytes: number; recordCount: number }>
   ```

## 存储

1. 音频文件：server 端 `dataDir/speech/audio/` 目录
2. 转写记录：全局 SQLite 数据库（非 project 级 DB）
3. History 页面显示总占用空间，提供手动清理
