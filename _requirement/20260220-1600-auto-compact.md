# 需求清单：Auto-Compact（自动对话压缩）
> 创建时间：2026-02-20 16:00
> 状态：已确认

## 功能需求

1. **自动 Compact**：当对话的 token 数超过 Agent 的 `compactThreshold` 时，自动执行 compact（调用模型生成对话摘要）
2. **消息过滤**：compact 后，发送给模型的不再是完整历史，而是 `[摘要消息] + [分界点之后的消息]`，从而降低 token 消耗
3. **多次 Compact**：同一对话可以触发多次 compact（长对话中会反复达到阈值）
4. **手动 Compact**：用户可以通过 UI 按钮手动触发 compact
5. **Compact 分界线 UI**：Chat 界面在 compact 发生的位置显示一个折叠式分界线，类似 tool call 的折叠显示风格
   - 默认折叠，显示 "Conversation Compacted" + token 信息
   - 点击展开可查看完整摘要内容
6. **历史消息不删除**：所有聊天记录在 UI 上完整可见，compact 只影响发送给模型的消息，不影响用户看到的内容
7. **Sub-agent 不需要 Compact**：Sub-agent 是单次 prompt 调用（无历史积累），不存在 compact 需求

## 技术约束

1. compact 摘要调用使用 `generateText`（非流式），因为用户不需要看到总结过程
2. 参考 Claude Code 的 compact prompt 设计（已提取到 `_design/20260220-auto-compact/claude-code-compact-prompts.md`）
3. 使用最近 assistant 消息的 `inputTokens` 作为 token 数判断依据
4. compact 记录存储在 SQLite 中（新增 `compact_records` 表）
5. 遵循现有项目架构模式：Service 接口层、HTTP/Mock 实现、drizzle-orm schema、programmatic migration

## 流程要求

1. compact 执行异步进行，不阻塞当前响应流
2. compact 后通过 SSE data event 通知前端
3. 消息过滤在 server 端 chat route 中完成（`convertToModelMessages` 之前）
