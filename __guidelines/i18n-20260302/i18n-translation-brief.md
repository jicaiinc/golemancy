# Golemancy i18n Translation Brief

> This document is the single source of truth for all translators working on Golemancy localization.

## 1. Product Context

**Golemancy** is a desktop application (Electron) for orchestrating autonomous AI agents. Users configure AI providers (OpenAI, Anthropic, Google, etc.), create projects, and build agent workflows that can execute tools, browse the web, and manage tasks.

The product uses a **pixel art / retro game aesthetic** (Minecraft-style). The brand metaphor is "Golems" — AI agents are magical constructs that users command.

**Target audience**: General AI users (not just developers). The UI should be accessible to non-technical users while still handling technical concepts (API keys, models, permissions).

## 3. Translation Principles

### 3.1 Tone
- **UI labels & buttons**: Concise, imperative. Prefer short words. Buttons should feel clickable.
- **Descriptions & hints**: Friendly, clear, guiding. Write as if teaching a new user.
- **Error messages**: Direct but not alarming (except security warnings, which should be deliberately cautious).
- **Onboarding**: Warm, welcoming, encouraging.
- **Dashboard/stats**: Data-dense, terse. Abbreviations are acceptable.

### 3.2 Preserve These Exactly
- `{{variable}}` interpolation placeholders — never translate, never add spaces inside braces
- `<a>`, `</a>` and other HTML/React tags
- Unicode symbols: `\u25BC` (▼), `\u2192` (→), `\u26A0` (⚠), `\u00B7` (·)
- Bracket formatting like `[{{running}}/{{total}} tools]`
- Prefix symbols like `+`, `<`, `<<` in button text
- `\n` newline characters
- File extensions (`.md`, `.zip`, `.json`)
- Unit suffixes: `ms`, `MB`, `KB`

### 3.3 Plural Forms
i18next uses CLDR plural rules. Keys ending in `_one` and `_other` must both be translated. Some languages (e.g., Polish, Czech, Arabic) have more plural forms — add `_few`, `_many` as needed per CLDR rules for that language.

Example (English):
```json
"entryCount_one": "{{count}} memory",
"entryCount_other": "{{count}} memories"
```

### 3.4 Creative Localization
Some strings are playful or cultural references. Adapt them naturally rather than translating literally:
- `"A wild error appeared!"` — Pokemon reference. Use an equivalent pop-culture joke or just a friendly error tone.
- `"Command Your AI Golems"` — Brand slogan. Translate the meaning ("command/control your AI agents") while keeping it catchy.
- `"Something Exploded"` — Error boundary title. Keep the playful pixel-game vibe.

### 3.5 Consistency Across Namespaces
These terms appear in multiple files and MUST use the same translation everywhere:

| English | Appears in | Context |
|---------|-----------|---------|
| Main Agent | agent, chat, project | The default agent that handles new conversations |
| Restricted / Sandbox / Unrestricted | nav, permissions, mcp, agent | Three security permission modes |
| Skill(s) | agent, skill, nav | Prompt templates assigned to agents |
| Sub-Agent(s) | agent | Delegated child agents |
| Conversation Tasks | nav, task, chat | Tasks created by agents during chat |
| Automations | nav, cron | Scheduled agent runs (NOT "Cron Jobs") |
| Artifacts | nav, workspace | Files generated by agents |
| Memory Bank | nav, memory | Persistent agent knowledge store |
| Provider | settings, onboarding, speech | AI API provider |
| Compact / Compacting | nav, chat | Context window compaction |
| OK / Failed / Testing... | settings, onboarding, speech, mcp | Connection test status |

## 4. Namespace Context Guide

### common (52 keys) — Shared vocabulary
Application-wide buttons, status labels, time formatting, confirmation dialogs, empty states. Every shared button ("Save", "Cancel", "Delete") lives here. Other namespaces reference these via `common:button.*`.

### chat (45 keys) — Chat experience
Conversation sidebar, message window, input box (with image/audio attachment), tool call display, compaction info. Note: `empty.configureText` contains `<a>` tags that must be preserved. "in/out" in token contexts means input/output tokens, not directions.

### nav (34 keys) — Navigation & status bar
Sidebar menu items, status bar (permission mode, context window, token stats). `item.artifacts` = "Artifacts" (agent-generated files, NOT "工件"). `item.tasks` = "Conversation Tasks" (agent-created, not user project management).

### error (50 keys) — Error messages
Three tones: (1) friendly generic fallback, (2) playful crash screen (ErrorBoundary), (3) technical server error codes. Server error keys match backend enum names (e.g., `AGENT_NOT_FOUND`).

### agent (86 keys) — Agent management (LARGEST)
Agent list, detail page (5 tabs), creation, topology view. Labels are uppercase ("SYSTEM PROMPT", "MODEL CONFIG"). Tool descriptions explain what each built-in tool does. `currentMain` refers to the project's Main Agent.

### project (33 keys) — Project management
Project list, creation, settings. "Main Agent" concept is shared with agent namespace.

### settings (50 keys) — Global settings
Provider management (add/edit/delete/test), appearance, language, default model, about info. `about.version` contains the brand tagline.

### permissions (76 keys) — Security configuration
Permission modes, sandbox rules (filesystem/network/commands). Security warnings should feel serious. `modal.unrestrictedBody` lists specific risks. Path examples (`~/.ssh`, `/etc`) should stay as-is.

### onboarding (71 keys) — First-run wizard
5-step setup: Welcome → Provider → Speech → Project → Complete. Warmest tone in the app. Brand slogan appears here. Provider/Speech steps mirror settings namespace patterns.

### dashboard (52 keys) — Analytics & monitoring
Token usage charts, time range filters, summary stats, runtime activity. Data-dense labels. Middle dot (`·`) is used as separator in stats strings.

### mcp (44 keys) — MCP server management
Server list, import, testing, form. Security warnings about third-party servers. `import.invalidFormat` includes a JSON structure hint that should be preserved as-is.

### cron (42 keys) — Automations (scheduled tasks)
User-facing name is "Automations", NOT "Cron Jobs". Cron expression presets provide friendly shortcuts ("Every 5 min", "Daily 9am"). Run status: success/error/running.

### skill (23 keys) — Skill templates
Skills are markdown-based prompt templates. Import supports `.md` and `.zip`. Deletion blocked if assigned to agents.

### memory (17 keys) — Memory bank
Persistent knowledge entries with content, source (agent name), and tags. Simplest namespace.

### task (13 keys) — Conversation tasks
Tasks created by agents during chat. Status: pending / in progress / completed / deleted. NOT project management tasks.

### workspace (19 keys) — Artifacts browser
File tree and preview. "Artifacts" = agent-generated files. `deleteModal.confirmPrefix` is a sentence prefix — the filename is appended after it. `preview.download` abbreviated as "DL".

### speech (57 keys) — Speech-to-text
Provider config, connection testing, transcription history, audio player, storage usage. Same test-status pattern as settings/onboarding. `clearAll.confirmWithFree` interpolates both count and file size.

Each file must be valid JSON. Use UTF-8 encoding. Match the exact key structure of the English source — do not add, remove, or rename keys.

## 6. Quality Checklist

After translation, verify:
- [ ] All `{{variables}}` preserved exactly
- [ ] All HTML tags (`<a>`, `</a>`) preserved
- [ ] All Unicode symbols preserved
- [ ] Plural forms correct for target language (add `_few`/`_many` if CLDR requires)
- [ ] Cross-namespace terms are consistent (see Section 3.5)
- [ ] No trailing commas in JSON (causes parse errors)
- [ ] No unescaped double quotes inside string values
- [ ] Brand terms kept in English (see Section 2)
- [ ] Tone matches namespace context (see Section 4)
